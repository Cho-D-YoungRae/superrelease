# superrelease v0.1.0 자기 dogfooding 구현 계획

> **실행 모델(중요):** 이 계획은 대부분 **운영(릴리스 실행)**이다. Task 1~2만 파일 산출물(그중 self-render 테스트는 TDD)이고, Task 3~5는 부작용이 있는 **런북**이다 — SDD 서브에이전트로 위임하지 않고 **인라인 실행**하며, 각 부작용(파일 수정·커밋·push·PR 생성/머지·태그 push·release 생성)마다 **dry-run 프리뷰 → 사용자 확인 → 실행** 게이트를 지킨다. 진행 추적용 체크박스(`- [ ]`) 유지.

**Goal:** superrelease가 자기 툴킷으로 자신을 v0.1.0으로 릴리스하고(release-pr 경로·GitHub Release), 커밋된 툴킷이 asset 템플릿에서 드리프트하지 못하게 self-render 테스트로 고정한다.

**Architecture:** `.superrelease/config.json`을 저작 → `render.py`로 `.superrelease/`+`.claude/skills/` 툴킷 렌더(2 PR 중 PR1=adopt) → 생성된 `release` 스킬로 release-pr 릴리스(PR2) → 머지 후 태그·Release 재개. 버전 소스는 `.claude-plugin/plugin.json`(json-path, 이미 0.1.0).

**Tech Stack:** Python 3.9+ 표준 라이브러리(render.py·version.py·unittest), git, gh CLI.

## Global Constraints

- **버전 소스**: `.claude-plugin/plugin.json`의 `version`(json-path). 이미 `0.1.0`, 태그 0개 → **첫 릴리스**. `version.py set 0.1.0`은 no-op.
- **config**: 아래 §Task 1의 전문 그대로. `release_pr_app` 골든에 가장 가까우며 render.py `--check`/실렌더로 **validate 통과 확인 완료**(exit 0, 7파일).
- **렌더 산출 7파일**(이 config에서): `.claude/skills/{release,release-notes}/SKILL.md`, `.superrelease/scripts/{version,next-version}.py`, `.superrelease/templates/{notes-single,changelog-entry,release-pr-body}.md`. `.github/release.yml`·hotfix·backfill·train·changed-packages·notes-package/train은 **skipped**(해당 config 아님).
- **마커**: 렌더 산출물에 `generated by superrelease v0.1.0` 마커가 박힌다(plugin_version 유래). 커밋본·self-render 모두 0.1.0이라 일치. (미래 버전 bump 시 마커가 갱신되므로 그 릴리스에서 툴킷 재렌더 필요 — §리스크.)
- **자립성**: 생성 툴킷은 `.superrelease/`·`.claude/` 상대경로만 참조(플러그인 경로 금지). 렌더로 자동 보장됨.
- **결정론**: self-render는 반드시 `superrelease`명 디렉터리 + `GIT_CEILING_DIRECTORIES`로 렌더(그러지 않으면 `project.name`이 달라져 오탐). 실렌더(`--repo .`)는 git remote basename="superrelease"라 동일.
- **부작용 게이트**: dry-run → 확인 → 실행. **push된 태그는 되돌리지 않는다**. 태그 push 직전 `git ls-remote --tags origin v0.1.0`가 비어 있어야 함.
- **gh 계정**: 이 레포 대상 gh 작업 직전 `gh auth switch --user Cho-D-YoungRae`, 완료 후 `gh auth switch --user aims-yrcho` 복원.
- **테스트**: 루트에서 `python3 -m unittest discover -s tests -q`. 단일: `cd tests && python3 -m unittest test_dogfood_selfrender -v; cd ..` (`python3 -m unittest tests.<mod>`는 `__init__.py` 없어 실패).
- **검증**: 랜딩 전 `claude plugin validate . --strict` PASS.
- **커밋**: Conventional Commits, 메시지 끝에 `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`.

- [ ] **Step 2: self-render 테스트 작성** — `tests/test_dogfood_selfrender.py`에 아래 전문:

```python
"""Dogfooding conformance: the committed toolkit must equal a fresh self-render.

superrelease renders its own release toolkit into this repo (.superrelease/ +
.claude/skills/). This re-renders that toolkit from the committed config and
asserts every committed file is byte-identical — so editing an asset template
without re-rendering (or hand-editing a rendered file) fails here immediately.
"""
import os
import subprocess
import sys
import tempfile
import unittest
from pathlib import Path

from helpers import ASSETS, PLUGIN_SCRIPTS, ROOT, write

CONFIG = ROOT / ".superrelease" / "config.json"
# self-render로 관리되는 툴킷 서브트리
TOOLKIT_DIRS = (".claude/skills", ".superrelease/scripts", ".superrelease/templates")
NOW = "2026-07-16T00:00:00+00:00"


def _is_source(path):
    return "__pycache__" not in path.parts and path.suffix != ".pyc"


class DogfoodSelfRenderTest(unittest.TestCase):
    def _self_render(self, tmp):
        # 디렉터리명을 superrelease로 고정 → project.name 결정론(git remote 무관)
        repo = Path(tmp) / "superrelease"
        (repo / ".superrelease").mkdir(parents=True)
        write(repo / ".superrelease" / "config.json",
              CONFIG.read_text(encoding="utf-8"))
        proc = subprocess.run(
            [sys.executable, str(PLUGIN_SCRIPTS / "render.py"),
             "--config", str(repo / ".superrelease" / "config.json"),
             "--assets", str(ASSETS), "--repo", str(repo), "--now", NOW],
            capture_output=True, text=True,
            env={**os.environ, "GIT_CEILING_DIRECTORIES": str(Path(tmp))})
        # returncode 0 = 커밋된 config가 validate_config를 통과함을 증명
        self.assertEqual(proc.returncode, 0, proc.stderr)
        out = {}
        for f in sorted(p for p in repo.rglob("*") if p.is_file() and _is_source(p)):
            rel = f.relative_to(repo).as_posix()
            if rel == ".superrelease/config.json":
                continue
            out[rel] = f.read_text(encoding="utf-8")
        return out

    def _committed_toolkit(self):
        out = {}
        for d in TOOLKIT_DIRS:
            base = ROOT / d
            for f in sorted(p for p in base.rglob("*") if p.is_file() and _is_source(p)):
                out[f.relative_to(ROOT).as_posix()] = f.read_text(encoding="utf-8")
        return out

    def test_committed_toolkit_matches_self_render(self):
        with tempfile.TemporaryDirectory() as tmp:
            rendered = self._self_render(tmp)
        committed = self._committed_toolkit()
        self.assertEqual(
            sorted(committed.keys()), sorted(rendered.keys()),
            "committed toolkit set != self-render set — re-render:\n"
            "  python3 skills/init/scripts/render.py --config "
            ".superrelease/config.json --assets skills/init/assets --repo .")
        for rel in rendered:
            self.assertEqual(committed[rel], rendered[rel], rel)


if __name__ == "__main__":
    unittest.main()
```
